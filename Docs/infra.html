infra/

  └── sqlserver/  ← باید ساخته شود
powershell
mkdir infra\postgres
mkdir infra\redis
mkdir infra\sqlserver
bash
mkdir -p infra/{postgres,redis,sqlserver}

تنظیم فایل Hosts

Nginx برای کار با ساب‌دامین‌های محلی نیاز دارد سه آدرس زیر به 127.0.0.1 resolve شوند:

www.localhost → 127.0.0.1
api.localhost → 127.0.0.1
localhost → 127.0.0.1
powershell — run as administrator
Add-Content -Path "C:\Windows\System32\drivers\etc\hosts" `
  -Value "`n127.0.0.1 localhost www.localhost api.localhost"
تأیید
Get-Content "C:\Windows\System32\drivers\etc\hosts" | Select-String "localhost"
bash
sudo bash -c 'echo "127.0.0.1 localhost www.localhost api.localhost" >> /etc/hosts'
ℹ️
اگر localhost قبلاً در فایل hosts بود، مشکلی نیست — خط جدیدی اضافه می‌شود. ویندوز اولین تطابق را استفاده می‌کند.
🔒

ساخت گواهی SSL محلی

⚠️
نام فایل‌ها مهم است! فایل‌ها باید دقیقاً privkey.pem و fullchain.pem نام داشته باشند — چون nginx/conf.d/default.conf دقیقاً همین نام‌ها را انتظار دارد.

ابتدا پوشه ssl را بسازید، سپس یکی از روش‌های زیر را انتخاب کنید:

powershell — روش پیشنهادی
# ساخت پوشه ssl
mkdir infra\nginx\ssl

# پاکسازی گواهی‌های قدیمی (اگه هستن)
Remove-Item -Force infra\nginx\ssl\*.pem -ErrorAction SilentlyContinue

# تولید گواهی با SAN برای همه ساب‌دامین‌ها
& "C:\Program Files\Git\usr\bin\openssl.exe" req -x509 -nodes `
  -days 365 `
  -newkey rsa:2048 `
  -sha256 `
  -keyout infra\nginx\ssl\privkey.pem `
  -out infra\nginx\ssl\fullchain.pem `
  -subj "/C=IR/ST=Tehran/L=Tehran/O=Microservice-Test/OU=Dev/CN=localhost" `
  -addext "subjectAltName=DNS:localhost,DNS:www.localhost,DNS:api.localhost,IP:127.0.0.1"
اگه مسیر OpenSSL متفاوت است
where.exe openssl
powershell — اگه OpenSSL پیدا نشد
# ساخت پوشه ssl
mkdir infra\nginx\ssl
Remove-Item -Force infra\nginx\ssl\*.pem -ErrorAction SilentlyContinue

docker run --rm -v "${PWD}/infra/nginx/ssl:/out" alpine/openssl req -x509 -nodes `
  -days 365 `
  -newkey rsa:2048 `
  -sha256 `
  -keyout /out/privkey.pem `
  -out /out/fullchain.pem `
  -subj "/C=IR/ST=Tehran/L=Tehran/O=Microservice-Test/OU=Dev/CN=localhost" `
  -addext "subjectAltName=DNS:localhost,DNS:www.localhost,DNS:api.localhost,IP:127.0.0.1"
bash
# ساخت پوشه ssl
mkdir -p infra/nginx/ssl

openssl req -x509 -nodes -days 365 \
  -newkey rsa:2048 \
  -sha256 \
  -keyout infra/nginx/ssl/privkey.pem \
  -out infra/nginx/ssl/fullchain.pem \
  -subj "/CN=localhost" \
  -addext "subjectAltName=DNS:localhost,DNS:www.localhost,DNS:api.localhost"

تأیید تولید گواهی:

powershell
Get-ChildItem infra\nginx\ssl\
bash
ls -lh infra/nginx/ssl/
✅
نتیجه مورد انتظار باید دو فایل ببینید: fullchain.pem (~۱.۴ KB) و privkey.pem (~۱.۷ KB)
🛡️

Trust کردن گواهی در ویندوز

ℹ️
این مرحله فقط برای ویندوز است. بدون آن، مرورگر هشدار «unsafe» نمایش می‌دهد.

قدم اول: پاک‌سازی گواهی‌های قدیمی localhost (اگه قبلاً تست کردی)

powershell — run as administrator
Get-ChildItem Cert:\LocalMachine\Root | Where-Object { $_.Subject -like "*localhost*" } | Remove-Item -Force
Get-ChildItem Cert:\CurrentUser\Root | Where-Object { $_.Subject -like "*localhost*" } | Remove-Item -Force -ErrorAction SilentlyContinue

قدم دوم: ایمپورت گواهی جدید

powershell — همان پنجره admin
cd "PATH\TO\PROJECT"
Import-Certificate -FilePath infra\nginx\ssl\fullchain.pem -CertStoreLocation Cert:\LocalMachine\Root

قدم سوم: تأیید ایمپورت

powershell
Get-ChildItem Cert:\LocalMachine\Root | Where-Object { $_.Subject -like "*Microservice-Test*" }
✅
نتیجه مورد انتظار باید یک گواهی با Subject شامل CN=localhost, O=Microservice-Test ببینید.
⚙️

تنظیم دامنه در Nginx Config

مقدار PLACEHOLDER-example.com را با دامنه محلی‌تان جایگزین کنید.

powershell
cd "PATH\TO\PROJECT"

(Get-Content .\infra\nginx\conf.d\default.conf) `
  -replace 'PLACEHOLDER-example.com', 'localhost' |
  Set-Content .\infra\nginx\conf.d\default.conf
bash
cd /path/to/project

# برای localhost
sed -i 's/PLACEHOLDER-example.com/localhost/g' infra/nginx/conf.d/default.conf

# برای localtest.me (جایگزین)
sed -i 's/PLACEHOLDER-example.com/localtest.me/g' infra/nginx/conf.d/default.conf

مایگریشن و ساخت Superuser

بعد از بالا آمدن کانتینرها، برای هر سرویس migrate و superuser بسازید:

📦 product-service
bash
docker exec product-service python manage.py migrate
docker exec -it product-service python manage.py createsuperuser
🔗 localhost:8001/admin/
🛒 cart-service
bash
docker exec cart-service python manage.py migrate
docker exec -it cart-service python manage.py createsuperuser
🔗 localhost:8002/admin/

تست اتصال

آدرس‌های قابل تست در مرورگر:

🔒 https://localhost 🔒 https://www.localhost 🔒 https://api.localhost 🔗 :8001/admin/ 🔗 :8002/admin/
⚠️
به دلیل Self-Signed بودن SSL، مرورگر هشدار امنیتی نمایش می‌دهد. روی Advanced و سپس Proceed to localhost کلیک کنید. اگر مرحله Trust را انجام داده‌اید، این هشدار نمایش داده نمی‌شود.

تست از ترمینال:

powershell
curl.exe -k --tls-max 1.2 https://localhost
curl.exe -k --tls-max 1.2 https://api.localhost
bash
curl -k https://localhost
curl -k https://api.localhost

تست سرویس‌های داخلی:

bash
docker exec gateway wget -qO- http://localhost:8080

docker exec product-service python -c \
  "import urllib.request; print(urllib.request.urlopen('http://localhost:8000/admin/').status)"
⏹️

توقف

توقف سرویس‌ها

docker compose --profile infra down -v

با استفاده از -v تمام داده‌های دیتابیس حذف می‌شوند. این عمل برگشت‌پذیر نیست.

عیب‌یابی

خطا راه‌حل
schannel: server closed abruptly از curl.exe -k --tls-max 1.2 استفاده کنید
nginx: emerg: cannot load certificate مرحله ساخت SSL را انجام دهید و نام فایل‌ها را بررسی کنید
.env: not found دستور cp .env.example .env را اجرا کنید
پنل ادمین Django باز نمی‌شود پورت را Publish کنید و Superuser بسازید
api.localhost resolve نمی‌شود فایل hosts را بررسی و ذخیره کنید
خطای dotnet sdk هنگام Build دستور docker builder prune را اجرا کنید
کانتینر فوراً متوقف می‌شود docker logs <container-name> را بررسی کنید
Frontend صفحه خالی نمایش می‌دهد مسیر /app/dist را در Dockerfile بررسی کنید
pip WARNING: Running as root این هشدار غیربحرانی است — در Docker طبیعی است
مرورگر همچنان هشدار SSL می‌دهد گواهی قدیمی را پاک کنید، گواهی جدید را import کنید و مرورگر را ریستارت کنید
💡
دیباگ عمومی برای دیدن لاگ یک کانتینر خاص: docker logs -f <container-name>
برای دیدن وضعیت همه کانتینرها: docker compose ps
