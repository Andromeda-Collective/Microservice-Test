مرحله‌ی ۱: تولید گواهی self-signed
این گواهی برای سه دامنه‌ی لوکال هست: localhost, www.localhost, api.localhost.

روش A: با OpenSSL خود Git (پیشنهادی)
PowerShell رو باز کن (نیازی به admin نیست) و در ریشه‌ی پروژه اجرا کن:

powershell

cd "PATH\TO\PROJECT"   # مثلاً I:\Tuxify\Andromeda\Microservice-infra\Microservice-infra-3

# پاک کردن گواهی‌های قبلی (اگه هستن)
Remove-Item -Force infra\nginx\ssl\*.pem -ErrorAction SilentlyContinue
Remove-Item -Force infra\nginx\ssl\*.key -ErrorAction SilentlyContinue
Remove-Item -Force infra\nginx\ssl\*.crt -ErrorAction SilentlyContinue

# تولید گواهی جدید
& "C:\Program Files\Git\usr\bin\openssl.exe" req -x509 -nodes `
  -days 365 `
  -newkey rsa:2048 `
  -sha256 `
  -keyout infra\nginx\ssl\privkey.pem `
  -out infra\nginx\ssl\fullchain.pem `
  -subj "/C=IR/ST=Tehran/L=Tehran/O=Microservice-Test/OU=Dev/CN=localhost" `
  -addext "subjectAltName=DNS:localhost,DNS:www.localhost,DNS:api.localhost,IP:127.0.0.1"
نکته‌ی مهم: نام فایل‌ها باید دقیقاً privkey.pem و fullchain.pem باشه — این‌ها دقیقاً همون چیزی هستن که infra/nginx/conf.d/default.conf انتظار داره. اگه نام دیگه‌ای بذاری، nginx ارور می‌ده.

روش B: با Docker (اگه OpenSSL پیدا نشد)
powershell

cd "PATH\TO\PROJECT"
Remove-Item -Force infra\nginx\ssl\*.pem -ErrorAction SilentlyContinue

docker run --rm -v "${PWD}/infra/nginx/ssl:/out" alpine/openssl req -x509 -nodes `
  -days 365 `
  -newkey rsa:2048 `
  -sha256 `
  -keyout /out/privkey.pem `
  -out /out/fullchain.pem `
  -subj "/C=IR/ST=Tehran/L=Tehran/O=Microservice-Test/OU=Dev/CN=localhost" `
  -addext "subjectAltName=DNS:localhost,DNS:www.localhost,DNS:api.localhost,IP:127.0.0.1"
تأیید تولید گواهی
powershell

Get-ChildItem infra\nginx\ssl\
باید این دو فایل رو ببینی:

text

fullchain.pem   (~1.4 KB)
privkey.pem     (~1.7 KB)
مرحله‌ی ۲: trust کردن گواهی در ویندوز
بدون این مرحله، گواهی تولید شده کار می‌کنه ولی مرورگر هشدار «unsafe» می‌ده.

۲-۱: پاک‌سازی گواهی‌های قدیمی localhost (اگه قبلاً تست کردی)
PowerShell رو Run as Administrator باز کن:

powershell

Get-ChildItem Cert:\LocalMachine\Root | Where-Object { $_.Subject -like "*localhost*" } | Remove-Item -Force
Get-ChildItem Cert:\CurrentUser\Root | Where-Object { $_.Subject -like "*localhost*" } | Remove-Item -Force -ErrorAction SilentlyContinue
۲-۲: import کردن گواهی جدید
همان PowerShell admin:

powershell

cd "PATH\TO\PROJECT"
Import-Certificate -FilePath infra\nginx\ssl\fullchain.pem -CertStoreLocation Cert:\LocalMachine\Root
خروجی باید چیزی شبیه این باشه:

text

   PSParentPath: Microsoft.PowerShell.Security\Certificate::LocalMachine\Root

Thumbprint                                Subject
----------                                -------
XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX  CN=localhost, OU=Dev, O=Microservice-Test, L=Tehran, S=Tehran, C=IR
۲-۳: تأیید import
powershell

Get-ChildItem Cert:\LocalMachine\Root | Where-Object { $_.Subject -like "*Microservice-Test*" }
باید یک گواهی ببینی.

مرحله‌ی ۳: تنظیم فایل hosts ویندوز
اگه قبلاً نکردی، این کار رو بکن تا api.localhost و www.localhost به 127.0.0.1 resolve بشن.

روش ساده:
PowerShell admin:

powershell

Add-Content -Path "C:\Windows\System32\drivers\etc\hosts" -Value "`n127.0.0.1 localhost www.localhost api.localhost"
بررسی:
powershell

Get-Content "C:\Windows\System32\drivers\etc\hosts" | Select-String "localhost"
باید این خط رو ببینی:

text

127.0.0.1 localhost www.localhost api.localhost
نکته: اگه localhost قبلاً توی فایل بود، مشکل نداره — خط جدیدی اضافه می‌شه. ویندوز اولین تطابق رو استفاده می‌کنه و همگی 127.0.0.1 هستن.
